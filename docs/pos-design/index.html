Each section follows the same shape: overview → key decisions → flow → logic/algorithms → edge cases → scaling. The Appendix contains reference tables used by the system.

Correctness first
  • Inventory deductions are atomic + logged.
  • Billing/refunds are auditable.
  • State transitions follow a strict FSM.
Operational clarity
  • Idempotency keys for all mutations.
  • Webhook ACK fast, process async.
  • Events after commit; consumers idempotent.
Scale path
  • Modular monolith → selective extraction.
  • Redis Pub/Sub now; Kafka when justified.
  • Async exports + aggregates for reporting.
This design is implementation-ready but intentionally technology-agnostic where choices don’t affect correctness. Adjust tax compliance, receipt format rules, and payment provider details to your region.
